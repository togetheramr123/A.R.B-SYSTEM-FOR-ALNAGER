# دليل تجهيز نظام ERP للعمل على Render بنسختين (Live & Demo)

هذا الدليل مخصص لك كمدير للنظام، يشرح لك بالخطوات كيف تقوم برفع السيستم على منصة Render وتجهيز النسختين (الأساسية والتجريبية).

---

## 1. تجهيز قواعد البيانات في Neon
بما أنك تستخدم **Neon PostgreSQL**، سنستغل ميزة رائعة وهي الـ (Branches):
1. افتح مشروعك في موقع Neon.
2. ادخل على **Branches** واضغط على `New Branch`.
3. سمه `erp-live` (هذه ستكون النسخة الحقيقية للشركة).
4. الـ Branch الحالي لديك سيبقى كما هو كنسخة تجريبية (يحتوي على كل داتا الاختبار الخاصة بك).
5. انسخ رابط الاتصال `DATABASE_URL` الخاص بـ `erp-live` وضعه في ملف جانبي.
6. انسخ رابط الاتصال `DATABASE_URL` الخاص بالـ Branch الأساسي (النسخة التجريبية) وضعه في ملف جانبي.

---

## 2. إنشاء التطبيقين في Render
سوف نقوم بإنشاء تطبيقين منفصلين (Web Services)، كلاهما يقرأ من نفس كود الـ GitHub الخاص بك:

### أ) النسخة الأساسية الحقيقية (ERP Live)
1. ادخل على Render واضغط `New +` ثم `Web Service`.
2. اختر مستودع الـ GitHub الخاص بالمشروع (ERP 2026).
3. اختر اسم التطبيق (مثال: `smart-erp-2026`).
4. الإعدادات الأساسية:
   - **Build Command:** `npm install && npx prisma generate && npm run build`
   - **Start Command:** `npm run start`
5. المتغيرات البيئية (Environment Variables):
   - أضف `DATABASE_URL` وضع فيه الرابط الخاص بفرع `erp-live` الذي جهزناه من Neon.
   - أضف `NEXT_PUBLIC_DEMO_URL` وضع فيه رابط النسخة التجريبية الذي سننشئه في الخطوة التالية.
6. اضغط `Create Web Service`.

### ب) النسخة التجريبية للموظفين (ERP Demo)
1. ادخل على Render مرة أخرى وأنشئ `Web Service` جديدة.
2. اختر نفس مستودع الـ GitHub.
3. اختر اسم مختلف (مثال: `smart-erp-demo`).
4. نفس الإعدادات الأساسية الخاصة بالبناء والتشغيل.
5. المتغيرات البيئية (Environment Variables) - **هنا يكمن الاختلاف**:
   - أضف `DATABASE_URL` وضع فيه رابط الـ Branch الأساسي القديم الذي يحتوي على الداتا التجريبية.
   - أضف متغير جديد: `NEXT_PUBLIC_IS_DEMO` واجعل قيمته `true` (هذا ما سيجعل الشريط الأحمر التحذيري يظهر للمستخدمين).
6. اضغط `Create Web Service`.

---

## 3. تصفير بيانات النسخة الأصلية
بمجرد أن يعمل تطبيق `ERP Live`، ستحتاج إلى تصفير بياناته لتسليمه للشركة نظيفاً (سيتم مسح الفواتير، والأصناف، والطلبيات، المخازن ... مع الاحتفاظ بالعملاء والموردين).

للقيام بذلك:
1. تأكد أنك تعمل على الفرع الجديد للـ Database الخاص بالنسخة الأصلية (`erp-live`).
2. يمكنك تشغيل السكربت الذي قمنا بإنشائه عبر التيرمنال باستخدام الأمر التالي:
   ```bash
   DATABASE_URL="رابط_النسخة_الأصلية" npx tsx scripts/zero_out_live_db.ts
   ```
   *(هذا الأمر سيمسح البيانات من النسخة الأصلية فقط، بينما ستظل النسخة التجريبية محتفظة بكل شيء).*

---

## كيف سيعمل النظام الآن؟
1. مدير الشركة سيدخل على رابط الـ Live العادي.
2. إذا أراد تدريب موظف جديد، سيضغط على زر **"الدخول للنسخة التجريبية (Demo)"** من صفحة تسجيل الدخول.
3. سيطلب منه السيستم **الرقم السري** (وهو 3000 بشكل افتراضي، ويمكن تغييره من إعدادات الشركة).
4. عند إدخال الرقم الصحيح، سيتم توجيهه فوراً إلى رابط الديمو (`smart-erp-demo.onrender.com`).
5. هناك سيرى الشريط الأحمر بأنه داخل منطقة التدريب، ويمكنه اللعب وتخريب البيانات كما يشاء دون أن يمس قاعدة البيانات الحقيقية للشركة!
6. إذا أردت يوماً تحديث السيستم برمجياً (مثلاً إضافة ميزة جديدة)، فقط قم بعمل `git push` لـ GitHub، وسيقوم Render تلقائياً بتحديث **كلا النسختين** في نفس اللحظة!
