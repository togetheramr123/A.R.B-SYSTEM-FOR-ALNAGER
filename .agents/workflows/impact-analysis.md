# قاعدة تحليل الأثر (Impact Analysis Guard)

## متى تُطبق هذه القاعدة؟
تُطبق **تلقائياً** عند كل طلب تعديل من المستخدم. لا تحتاج لأمر خاص لتفعيلها.

## ما الذي يجب فعله قبل تنفيذ أي تعديل؟

### الخطوة 1: تصنيف التعديل
صنّف الطلب إلى أحد الأنواع التالية:
- **بنّاء (Additive):** إضافة ميزة جديدة، تحسين أداء، إصلاح خطأ — ✅ نفذ مباشرة.
- **تكاملي (Integrative):** ربط مكونات ببعضها، توحيد واجهات — ✅ نفذ مباشرة.
- **هدّام محتمل (Potentially Destructive):** تغيير سلوك موجود، حذف كود، استبدال مكون، تغيير schema — ⚠️ يجب تحليل الأثر أولاً.

### الخطوة 2: تحليل الأثر (للتعديلات الهدّامة المحتملة فقط)
قبل تنفيذ أي تعديل هدّام محتمل، يجب:

1. **البحث عن الاعتماديات (Dependencies):** ابحث في الكود عن كل الملفات التي تستخدم أو تعتمد على المكون/الدالة/الحقل المراد تعديله.
2. **تقييم الأثر:** حدد بالضبط ما سيتأثر:
   - هل سيُحذف سلوك يعمل حالياً؟
   - هل سيتغير شكل واجهة مستخدم موجودة؟
   - هل ستتأثر دورة عمل (Business Cycle) مكتملة؟
   - هل سيتأثر schema قاعدة البيانات بشكل يتطلب migration؟
   - هل ستتعطل اختبارات موجودة؟

3. **إبلاغ المستخدم:** قبل التنفيذ، أعرض تقرير أثر مختصر بهذا الشكل:

```
⚠️ تحليل الأثر:
━━━━━━━━━━━━━━━━━
📌 التعديل المطلوب: [وصف مختصر]

🔴 سيتم إلغاء/إزالة:
   - [ميزة أو سلوك سيُحذف]

🟡 سيتأثر/يتغير:
   - [مكون أو صفحة ستتغير]

🟢 لن يتأثر:
   - [الأجزاء الآمنة]

💡 البديل المقترح: [إن وجد حل يحافظ على كل شيء]
```

4. **انتظار الموافقة:** لا تنفذ التعديل الهدّام إلا بعد موافقة صريحة من المستخدم.

### الخطوة 3: الحماية أثناء التنفيذ
- إذا وافق المستخدم، نفذ التعديل مع الحفاظ على الكود القديم كتعليق (comment) إذا كان هناك احتمال للرجوع إليه.
- شغّل الاختبارات بعد كل تعديل للتأكد أن لا شيء انكسر.

## أمثلة عملية

### مثال 1: المستخدم يطلب "احذف الحفظ التلقائي"
**التصنيف:** هدّام محتمل
**الأثر:** الحفظ التلقائي للأوامر الموجودة سيتوقف، مما يعني فقدان بيانات إذا أغلق المتصفح.
**الإبلاغ:** نعم — أوضح أن الأوامر الموجودة ستفقد ميزة الحفظ الصامت.
**البديل:** إيقاف التوجيه فقط مع الإبقاء على الحفظ الصامت.

### مثال 2: المستخدم يطلب "غيّر اسم حقل في الداتابيز"
**التصنيف:** هدّام محتمل
**الأثر:** كل الملفات التي تستخدم هذا الحقل ستنكسر.
**الإبلاغ:** نعم — أعرض قائمة بكل الملفات المتأثرة.
**البديل:** تعديل الحقل مع تحديث كل الملفات المعتمدة عليه.

### مثال 3: المستخدم يطلب "أضف زر جديد للطباعة"
**التصنيف:** بنّاء
**الأثر:** لا يوجد أثر سلبي.
**الإبلاغ:** لا حاجة — نفذ مباشرة.
