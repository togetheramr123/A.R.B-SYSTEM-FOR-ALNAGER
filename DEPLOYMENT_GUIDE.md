
# دليل مشاركة النظام مع الفريق (Deployment Guide)

لمشاركة النظام مع فريق المبيعات والإدارة، لديك خياران رئيسيان:

## الخيار 1: المشاركة المؤقتة (الأسرع - لا يحتاج تغييرات)
إذا كنت تريد عرض النظام **الآن** من جهازك مباشرة دون رفعه على سيرفر، استخدم خدمة **Ngrok**.

1. قم بتحميل وتثبيت [Ngrok](https://ngrok.com/download).
2. افتح سطر الأوامر (Terminal) واكتب:
   ```bash
   ngrok http 3000
   ```
3. سيظهر لك رابط (مثال: `https://abcd-123.ngrok-free.app`). أرسل هذا الرابط لفريقك وسيعمل النظام لديهم طالما جهازك يعمل.

---

## الخيار 2: الرفع على منصة مجانية (Vercel) - الحل الدائم
لرفع النظام بشكل دائم، نوصي باستخدام **Vercel** (للموقع) و **Neon/Supabase** (لقاعدة البيانات).

**ملاحظة هامة:** النظام حالياً يستخدم قاعدة بيانات محلية (SQLite) وهي ملف لا يعمل جيداً على السيرفرات السحابية المجانية. يجب تحويل قاعدة البيانات إلى Postgres.

### الخطوات:

1. **إنشاء قاعدة بيانات سحابية:**
   - سجل مجاناً في موقع [Neon.tech](https://neon.tech).
   - أنشئ مشروع جديد واحصل على رابط الاتصال (Connection String).

2. **تجهيز المشروع:**
   - في ملف `prisma/schema.prisma`، غير المزود:
     ```prisma
     datasource db {
       provider = "postgresql" // بدلاً من sqlite
       url      = env("DATABASE_URL")
     }
     ```
   - في ملف `.env`، ضع رابط قاعدة البيانات الجديد.

3. **الرفع على Vercel:**
   - نزل أداة Vercel CLI: `npm i -g vercel`
   - اكتب الأمر: `vercel`
   - اتبع الخطوات (Yes للكل).
   - اذهب لإعدادات المشروع في Vercel وأضف المتغير `DATABASE_URL`.

إذا أردت، يمكنني القيام بتحويل قاعدة البيانات لك الآن وتجهيزها للرفع. أخبرني "جهز النظام للرفع على Vercel".
